Plataforma ASP.NET

O que é ASP.NET?​

O ASP.NET é uma plataforma da Microsoft para desenvolvimento de aplicações web utilizando C#.

Ele fornece:

  • sistema de rotas;
  • processamento HTTP;
  • middlewares;
  • injeção de dependência;
  • autenticação;
  • APIs;
  • páginas web;
  • MVC.

Ou seja:

o ASP.NET fornece toda a infraestrutura necessária para criar aplicações web modernas.

ASP.NET MVC​

O que é uma aplicação web?​

Antes de entender o ASP.NET MVC, precisamos entender o que realmente é uma aplicação web.

Quando acessamos um site:

  • o navegador envia uma requisição;
  • um servidor recebe essa requisição;
  • o servidor processa os dados;
  • uma resposta é enviada de volta.

Essa resposta normalmente contém:

  • HTML;
  • CSS;
  • JavaScript;
  • dados.

Ou seja:

aplicações web funcionam através da comunicação entre cliente e servidor.

O que é um servidor web?​

Um servidor web é um programa que fica “escutando” requisições vindas da internet.

Quando alguém acessa:

https://meusite.com/produtos

o servidor recebe essa requisição e decide:

  • qual código executar;
  • quais dados buscar;
  • qual resposta retornar.

Em aplicações ASP.NET, esse servidor é criado pela própria aplicação.

Criação do Servidor Web com ASP.NET MVC​

// Instancia o construtor do servidor...
// ...configura as dependências internas (controladores e serviços) e externas (repositórios)
WebApplicationBuilder builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllersWithViews();

// Constroi a instância do servidor...
// ...configura os middlewares executados durante requisições
WebApplication app = builder.Build();

// Mapeia controladores e rotas que serão gerenciadas pelo servidor
app.UseStaticFiles();
app.UseRouting();
app.MapDefaultControllerRoute();

// Inicia o loop do servidor web, escuta por requisições na porta especificada
app.Run();

WebApplicationBuilder​

WebApplicationBuilder builder = WebApplication.CreateBuilder(args);

Aqui o ASP.NET começa a configurar o servidor.

Nesse momento configuramos:

  • serviços;
  • dependências;
  • controllers;
  • banco de dados;
  • autenticação.

Services​

builder.Services.AddControllersWithViews();

O ASP.NET utiliza um sistema chamado Injeção de Dependência.

Os serviços registrados aqui poderão ser utilizados em toda a aplicação.

Nesse caso estamos registrando:

  • controllers;
  • views;
  • funcionalidades do MVC.

Build​

WebApplication app = builder.Build();

Aqui a aplicação monta o pipeline interno do servidor.

Middlewares​

app.UseStaticFiles();
app.UseRouting();

Middlewares são etapas executadas durante uma requisição.

Cada middleware pode:

  • processar;
  • alterar;
  • bloquear;
  • encaminhar;

a requisição.

Rotas​

app.MapDefaultControllerRoute();

As rotas conectam URLs aos controllers.

Exemplo:

/produtos

pode ser direcionado para:

ProdutosController

Loop do Servidor​

app.Run();

Aqui o servidor começa a executar continuamente.

Ele permanece:

  • escutando requisições;
  • processando respostas;
  • atendendo usuários.
