Encapsulamento

Até agora, nosso sistema vem crescendo com:

  • classes (Caixa, Revista)
  • regras de validação
  • manipulação direta dos dados

Mas surge um novo problema:

"Estamos controlando corretamente o acesso aos dados das nossas classes?"

👉 Ou será que qualquer parte do sistema pode modificar tudo livremente?

Objetivo desta etapa​

Introduzir o conceito de:

  • Encapsulamento

E com isso:

👉 proteger os dados
👉 controlar alterações
👉 evitar estados inválidos
👉 deixar o código mais seguro e organizado

O conceito de Encapsulamento​

O que é encapsulamento?​

Encapsulamento significa:

esconder os detalhes internos de uma classe e controlar como os dados são acessados e modificados

👉 A classe passa a ser responsável por si mesma

Analogia simples​

Imagine uma caixa física:

  • você não mexe diretamente nos mecanismos internos
  • você usa interfaces externas (abrir, fechar, guardar)

👉 No código é a mesma ideia:

  • não acessar diretamente os dados
  • usar métodos controlados

Problema sem encapsulamento​

Veja a classe atual:

public class Caixa : EntidadeBase
{
    public string Etiqueta { get; set; } = string.Empty;
    public string Cor { get; set; } = string.Empty;
    public int DiasDeEmprestimo { get; set; } = 7;
}

👉 Qualquer lugar do sistema pode fazer:

caixa.DiasDeEmprestimo = -10;
  • Isso quebra a regra de negócio
  • O objeto fica inválido

Aplicando Encapsulamento​

A ideia é:

👉 controlar como os dados são alterados

Requisito 1: Encapsulando a classe Caixa​

public class Caixa : EntidadeBase
{
    public string Etiqueta { get; private set; } = string.Empty;
    public string Cor { get; private set; } = string.Empty;
    public int DiasDeEmprestimo { get; private set; } = 7;

    public Caixa(string etiqueta, string cor, int   diasDeEmprestimo)
    {
        AtualizarRegistro(etiqueta, cor, diasDeEmprestimo);
    }

    public void AtualizarRegistro(string etiqueta, string cor, int diasDeEmprestimo)
    {
        Etiqueta = etiqueta;
        Cor = cor;
        DiasDeEmprestimo = diasDeEmprestimo;
    }
}

O que mudou?​

Antes:​

  • qualquer código podia alterar os dados

Agora:​

👉 apenas a própria classe controla isso

Uso do private set​

public string Etiqueta { get; private set; }

👉 Isso significa:

  • pode ler de fora
  • mas só pode alterar dentro da class

Benefício direto​

Agora não é possível fazer isso:

caixa.DiasDeEmprestimo = -10; // ❌ erro

👉 Isso força o uso de métodos controlados

Requisito 2: Encapsulamento na Revista​

public class Revista : EntidadeBase
{
    public string Titulo { get; private set; }
    public int NumeroEdicao { get; private set; }
    public int AnoPublicacao { get; private set; }
    public Caixa Caixa { get; private set; }
    public StatusRevista Status { get; private set; }

    public Revista(string titulo, int numeroEdicao, int anoPublicacao, Caixa caixa)
    {
        AtualizarRegistro(titulo, numeroEdicao, anoPublicacao, caixa);
    }
}

Controle de estado​

Veja esses métodos:

public void Emprestar()
{
    Status = StatusRevista.Emprestada;
}

public void Devolver()
{
    Status = StatusRevista.Disponivel;
}

Sem encapsulamento​

Alguém poderia fazer:

revista.Status = StatusRevista.Emprestada;
  • sem regra
  • sem controle

Com encapsulamento​

👉 só é possível mudar o estado por métodos específicos

Regra importante​

Encapsulamento responde a pergunta:

"Quem é responsável por garantir que este dado é válido?"

👉 Resposta: a própria classe

Ligação com os outros pilares​

Encapsulamento + Herança​

  • Herança reaproveita estrutura
  • Encapsulamento protege essa estrutura

Encapsulamento + Polimorfismo​

  • Polimorfismo usa objetos de forma genérica
  • Encapsulamento garante que esses objetos estão válidos

Resultado no sistema​

Antes:

qualquer código podia alterar qualquer dado alto risco de erro

Agora:

  • 👉 classes controlam seus próprios dados
  • 👉 regras ficam centralizadas
  • 👉 código mais seguro

Resumo final​

Encapsulamento permite:

  • proteger dados internos
  • controlar alterações
  • garantir consistência
  • reduzir bugs
