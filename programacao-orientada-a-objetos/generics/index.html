Generics

Até aqui, já vimos como classes representam objetos, como usamos herança, polimorfismo e interfaces para organizar melhor o sistema.

Agora vamos avançar mais um passo na construção de código reutilizável.

Em muitos momentos do desenvolvimento, percebemos que estamos repetindo a mesma lógica para tipos diferentes.

Por exemplo:

  • um repositório de Caixa;
  • um repositório de Revista;
  • um repositório de Amigo.

Todos fazem praticamente a mesma coisa, mas para tipos diferentes.

É exatamente nesse tipo de situação que entram os Generics.

O que são Generics?​

Generics permitem que criemos classes, métodos ou estruturas que funcionam com tipos diferentes, sem precisar reescrever a mesma lógica várias vezes.

Em vez de criar uma versão para cada tipo, criamos uma versão genérica, que pode ser adaptada conforme necessário.

Veja um exemplo simples:

public class Caixa<T>
{
    public T Conteudo { get; set; }
}

Aqui estamos dizendo:

“Essa classe trabalha com algum tipo T, que será definido depois.”

O T é um tipo genérico, um espaço reservado para um tipo real.

Como isso funciona na prática?​

Ao usar a classe, escolhemos o tipo concreto:

Caixa<int> caixaNumero = new Caixa<int>();
caixaNumero.Conteudo = 10;

Ou:

Caixa<string> caixaTexto = new Caixa<string>();
caixaTexto.Conteudo = "Olá mundo";

A mesma classe funciona para tipos diferentes.

Ou seja:

  • a estrutura é a mesma;
  • o tipo muda conforme a necessidade.

Por que isso é útil?​

Sem generics, teríamos que criar várias versões da mesma classe:

  • CaixaDeNumero
  • CaixaDeTexto
  • CaixaDeRevista

Isso gera:

  • duplicação de código;
  • maior dificuldade de manutenção;
  • mais chances de erro.

Com generics, criamos uma única solução reutilizável.

Generics e Reutilização​

Esse é o principal benefício.

Generics permitem escrever código uma vez e reutilizá-lo para diferentes tipos.

Veja um exemplo mais próximo da realidade:

public class Repositorio<T>
{
    private List<T> registros = new List<T>();

    public void Cadastrar(T item)
    {
        registros.Add(item);
    }

    public List<T> SelecionarTodos()
    {
        return registros;
    }
}

Agora podemos fazer:

Repositorio<Revista> repositorioRevista = new();
Repositorio<Amigo> repositorioAmigo = new();

Repare:

  • o comportamento é o mesmo;
  • o tipo armazenado muda.

Generics e Tipagem Forte​

Outro ponto importante é a segurança.

Sem generics, muitas vezes precisaríamos usar object:

List<object> lista = new List<object>();

O problema disso é que perdemos a segurança de tipos:

lista.Add(10);
lista.Add("texto");

Depois precisamos fazer cast:

int numero = (int)lista[0];

Isso pode gerar erros em tempo de execução.

Com generics:

List<int> lista = new List<int>();

Agora o compilador garante:

  • só entram int;
  • não precisamos de cast;
  • menos erros.

Generics e Performance​

Generics também ajudam na performance.

Quando usamos object, ocorre boxing/unboxing para tipos de valor:

object numero = 10; // boxing
int valor = (int)numero; // unboxing

Com generics, isso não acontece:

List<int> lista = new List<int>();

O tipo é tratado diretamente, sem conversões desnecessárias.

Constraints (Restrições de Tipo)​

Podemos restringir quais tipos podem ser usados no generic.

Exemplo:

public class Repositorio<T> where T : class
{
}

Aqui estamos dizendo:

“T precisa ser um tipo de referência.”

Outro exemplo:

public class Servico<T> where T : IEntidade
{
}

Agora o tipo precisa implementar uma interface específica.

Isso é importante porque permite:

  • garantir comportamentos;
  • acessar métodos do tipo genérico com segurança.

Generics e Métodos​

Generics não servem apenas para classes.

Também podemos usá-los em métodos:

public T ObterPrimeiro<T>(List<T> lista)
{
    return lista[0];
}

Aqui o método funciona com qualquer tipo de lista.

Generics e Abstração​

Generics ajudam a trabalhar em um nível mais abstrato.

Em vez de pensar:

“Essa classe funciona para revista”

passamos a pensar:

“Essa classe funciona para qualquer tipo que respeite essa estrutura”

Isso deixa o código:

  • mais flexível;
  • mais reutilizável;
  • mais organizado.

Generics não substituem tipos concretos​

É importante entender:

  • generics não eliminam classes concretas;
  • eles tornam essas classes mais reutilizáveis.

O tipo concreto ainda existe:

Repositorio<Revista>

Mas agora ele é apenas uma especialização de uma estrutura genérica.

Conexão com orientação a objetos​

Generics trabalham junto com os outros conceitos:

  • com classes, permitem reutilização;
  • com interfaces, garantem contratos;
  • com polimorfismo, aumentam a flexibilidade.

Eles não substituem esses conceitos — eles complementam.

Conclusão​

Generics permitem criar estruturas reutilizáveis que funcionam com diferentes tipos.

Em resumo:

  • evitam duplicação de código;
  • aumentam a segurança de tipos;
  • melhoram a performance;
  • deixam o código mais flexível.

Ao usar generics, passamos a pensar menos em tipos específicos e mais em comportamentos reutilizáveis.

E esse é um passo importante para escrever código mais limpo, organizado e orientado a objetos.
