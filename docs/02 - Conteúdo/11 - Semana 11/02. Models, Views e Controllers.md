## O que significa MVC?

MVC significa:

- Model
- View
- Controller

Esse padrão separa responsabilidades dentro da aplicação.

## Separação de Responsabilidades

O principal objetivo do MVC é organizar melhor a aplicação.

Sem MVC, seria comum misturar:

- HTML;
- acesso ao banco;
- regras de negócio;
- processamento HTTP.

Isso deixaria o sistema difícil de manter.

Com MVC:

- Models cuidam dos dados;
- Views cuidam da interface;
- Controllers coordenam o fluxo.

## Model

O Model representa os dados e regras de negócio.

Exemplo:

```csharp
public class Produto
{
    public string Nome { get; set; }
    public decimal Preco { get; set; }
}
```

O model normalmente contém:

- entidades;
- validações;
- regras;
- dados do sistema.

## View

A View representa a interface exibida para o usuário.

Ela é responsável pela apresentação visual.

Exemplo:

```html
<h1>@Model.Nome</h1>
<p>@Model.Preco</p>
```

## Controller

O Controller é o intermediário entre:

- usuário;
- regras;
- interface.

Ele recebe requisições HTTP:

```
GET /produtos
```

Processa a lógica necessária e retorna uma resposta.

Exemplo:

```csharp
public class ProdutosController : Controller
{
    public IActionResult Index()
    {
        return View();
    }
}
```

## Fluxo de uma requisição

Uma requisição normalmente funciona assim:

1. O navegador envia uma requisição;
2. O servidor ASP.NET recebe;
3. O sistema de rotas encontra o controller correto;
4. O controller executa a lógica;
5. A view é renderizada;
6. O HTML retorna para o navegador.

Para implementação desse fluxo, o MVC ajuda a manter o código:

- mais organizado;
- mais reutilizável;
- mais desacoplado;
- mais fácil de manter.
