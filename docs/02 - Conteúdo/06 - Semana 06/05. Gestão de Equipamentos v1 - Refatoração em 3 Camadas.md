Nesta etapa, vamos evoluir o sistema de Gestão de Equipamentos para uma nova forma de organização: arquitetura em camadas.

Na versão anterior, todo o código estava concentrado em um único fluxo sequencial. Isso funcionava, mas misturava responsabilidades diferentes no mesmo lugar.

Agora, o objetivo é separar o sistema em partes bem definidas, cada uma com uma função específica.

O problema da versão anterior

## Na versão anterior, o programa fazia tudo no mesmo lugar:

- exibia o menu;
- capturava dados do usuário;
- armazenava dados em memória;
- realizava buscas e alterações.

Tudo isso acontecia dentro do `Main`.

Isso funciona para sistemas pequenos, mas conforme o sistema cresce, o código começa a ficar:

- difícil de entender;
- difícil de manter;
- difícil de modificar sem quebrar algo.

## A ideia da refatoração

Para resolver isso, vamos dividir o sistema em três camadas:

### 1. Domínio

Responsável por representar os dados e regras do sistema.

Aqui vivem as classes que representam o “mundo real”.

Exemplo:

- `Equipamento`

### 2. Infraestrutura de Dados

Responsável por armazenar e recuperar os dados.

Aqui vivem as classes que sabem “onde” e “como” os dados são guardados.

Exemplo:

- `RepositorioEquipamento`

### 3. Apresentação

Responsável por interagir com o usuário.

Aqui vivem as telas, menus e entradas de dados.

Exemplo:

- `TelaEquipamento`

## Requisito 1: Separando o Domínio

### O que mudou?

Na prática, quase nada muda aqui — e isso é intencional.

A classe Equipamento continua representando os dados:

```cs
public class Equipamento
{
    public string id;
    public string nome;
    public string fabricante;
    public decimal precoAquisicao;
    public DateTime dataFabricacao;
}
```

### Por que isso é importante?

Porque o domínio deve ser independente.

Ele não deve saber:

- se os dados vêm do console;
- se estão em um array;
- se estão em um banco de dados.

Ele apenas representa o objeto.

## Requisito 2: Camada de Infraestrutura de Acesso à Dados

### O que o sistema aprende a fazer?

Agora criamos uma classe responsável apenas por gerenciar os dados:

```cs
namespace GestaoDeEquipamentos.ConsoleApp.Infraestrutura;

public class RepositorioEquipamento
{
    public Equipamento?[] equipamentos = new Equipamento[100];

    public void Cadastrar(Equipamento novoEquipamento)
    {
        novoEquipamento.id = Convert
            .ToHexString(RandomNumberGenerator.GetBytes(20))
            .ToLower()
            .Substring(0, 7); // 0-255

        for (int i = 0; i < equipamentos.Length; i++)
        {
            Equipamento? e = equipamentos[i];

            if (e == null)
            {
                equipamentos[i] = novoEquipamento;
                break;
            }
        }
    }

    public bool Editar(string idSelecionado, Equipamento novoEquipamento)
    {
        Equipamento? equipamentoSelecionado = SelecionarPorId(idSelecionado);

        if (equipamentoSelecionado == null)
            return false;

        equipamentoSelecionado.nome = novoEquipamento.nome;
        equipamentoSelecionado.fabricante = novoEquipamento.fabricante;
        equipamentoSelecionado.precoAquisicao = novoEquipamento.precoAquisicao;
        equipamentoSelecionado.dataFabricacao = novoEquipamento.dataFabricacao;

        return true;
    }

    public bool Excluir(string idSelecionado)
    {
        for (int i = 0; i < equipamentos.Length; i++)
        {
            Equipamento? e = equipamentos[i];

            if (e == null)
                continue;

            if (e.id == idSelecionado)
            {
                equipamentos[i] = null;
                return true;
            }
        }

        return false;
    }

    public Equipamento? SelecionarPorId(string idSelecionado)
    {
        Equipamento? equipamentoSelecionado = null;

        for (int i = 0; i < equipamentos.Length; i++)
        {
            Equipamento? e = equipamentos[i];

            if (e == null)
                continue;

            if (e.id == idSelecionado)
            {
                equipamentoSelecionado = e;
                break;
            }
        }

        return equipamentoSelecionado;
    }

    public Equipamento?[] SelecionarTodos()
    {
        return equipamentos;
    }
}
```

### Qual é a responsabilidade dela?

- armazenar equipamentos;
- cadastrar novos;
- editar existentes;
- excluir;
- buscar por id;
- listar todos.

### Por que isso é importante?

Agora o sistema tem um lugar único responsável pelos dados.

Isso significa que:

- não precisamos mais repetir loops pelo sistema;
- mudanças futuras ficam centralizadas;
- o código fica mais organizado.

## O que essa versão consolida?

Agora o sistema está organizado em camadas:

1. `Domínio` → representa os dados
2. `Infraestrutura` → gerencia os dados
3. `Apresentação` → interage com o usuário

### Por que isso é importante?

Essa separação permite:

- código mais organizado;
- maior facilidade de manutenção;
- possibilidade de evolução.

Por exemplo:

- no futuro, poderíamos trocar o array por banco de dados
- ou trocar o console por interface web

E o restante do sistema continuaria funcionando.
